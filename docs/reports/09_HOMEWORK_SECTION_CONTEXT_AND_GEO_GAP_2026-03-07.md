# تقرير سياق الشعبة والموقع الجغرافي - 2026-03-07

## 1) المشكلة التي ظهرت في شاشة الواجبات

1. قائمة اختيار الشعبة كانت تُظهر اسمًا متكررًا مثل `الشعبة أ (A)` بدون سياق الصف.
2. السبب: الشعبة (`sections`) فريدة داخل الصف فقط، لذلك تتكرر عبر صفوف مختلفة.
3. النتيجة التشغيلية: المستخدم لا يعرف أي شعبة تخص أي صف عند إنشاء الواجب.

## 2) ما تم إصلاحه الآن

1. تم توحيد صيغة عرض الشعبة إلى:
   - `الصف/المرحلة (الكود) / الشعبة (الكود)`.
2. تم تطبيق ذلك في:
   - `frontend/src/features/homeworks/components/homeworks-workspace.tsx`
   - `frontend/src/features/student-homeworks/components/student-homeworks-workspace.tsx`
3. تم إضافة Utility موحد:
   - `frontend/src/lib/option-labels.ts` (`formatSectionWithGradeLabel`)
4. تم إضافة ملاحظة تشغيلية داخل شاشة الواجبات:
   - القاعة/مكان الحصة يُضبط من `الجدول الدراسي` وليس من تعريف الشعبة.

## 3) وضع القاعة/المبنى في النموذج الحالي

1. `Section` لا يحتوي حاليًا FK مباشر إلى مبنى/قاعة.
2. مكان الحصة الحالي يُمثل عبر:
   - `timetable_entries.roomLabel` (نصي)
3. يوجد Lookup للمباني:
   - `lookup_buildings`
4. لكن لا يوجد حتى الآن ربط مباشر من الشعبة أو الحصة إلى `lookup_buildings`.

## 4) وضع الموقع الجغرافي للطالب

1. Lookup جغرافي موجود (محافظة/مديرية/عزلة/قرية/محلة):
   - `governorates`, `directorates`, `sub_districts`, `villages`, `localities`
2. الربط المباشر بالطالب غير مكتمل حاليًا:
   - `students`: لا يحتوي حاليًا FK للموقع الجغرافي.
   - `guardians`: يحتوي نص سكن (`residenceText`) فقط بدون FK جغرافي.

## 5) نتيجة مراجعة توثيق نظام التعليم (System 05)

1. يوجد تفاوت بين بعض ملفات التوثيق القديمة والبنية البرمجية الحالية.
2. أمثلة تفاوت:
   - استخدام أسماء legacy مثل `classroom_id`, `semester_id` في بعض ملفات التوثيق.
   - بينما التنفيذ الحالي يعتمد `sectionId`, `academicTermId`.
3. هذا التفاوت يسبب لبسًا عند مطابقة الشاشة مع التوثيق.

## 6) التوصية التنفيذية التالية

1. تنفيذ Migration موحدة للموقع الجغرافي:
   - إضافة `localityId` (اختياري) على `students` و`guardians`.
   - الإبقاء على حقل النص (`residenceText`/`addressText`) كحقل وصفي.
2. تحديث DTO + Services + Frontend forms لاختيار `locality`.
3. لاحقًا (اختياري): إضافة كيان صريح للقاعات/الغرف مع ربط بالمبنى بدل `roomLabel` النصي.
