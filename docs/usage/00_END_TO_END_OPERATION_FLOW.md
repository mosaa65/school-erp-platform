# تدفق التشغيل الكامل (E2E)

هذا الملف يوضح التسلسل التشغيلي الموصى به لتشغيل النظام واختبار سيناريو كامل من الإعدادات حتى النتائج.

## 1) تجهيز البيئة

1. شغّل قاعدة البيانات MySQL.
2. داخل `backend`:
   1. `npm install`
   2. `npm run prisma:generate`
   3. `npm run prisma:migrate:deploy`
   4. `npm run prisma:seed:core`
   5. `npm run start:dev`
3. داخل `frontend`:
   1. `npm install`
   2. `npm run dev`

## 2) تسجيل الدخول

1. ادخل من واجهة الويب على `http://localhost:3001`.
2. استخدم حساب الـadmin الناتج من seed الأساسي.
3. تأكد أن القائمة الجانبية تعرض مجموعات الأنظمة (01 إلى 05).

## 3) التسلسل الوظيفي المعتمد

1. System 01: بناء الهوية المؤسسية (المستخدمون، الأدوار، الصلاحيات، المرجعيات).
2. System 02: تعريف الهيكل الأكاديمي (سنوات، فصول، صفوف، مواد، شعب).
3. System 03: إدخال موظفين وربط مهام/تدريس.
4. System 04: إدخال الطلاب وأولياء الأمور والقيود الدراسية.
5. System 05: تنفيذ التقييمات والدرجات واستخراج النتائج.

## 4) أوامر الجودة قبل التسليم

1. داخل `backend`: `npm run build`
2. داخل `frontend`:
   1. `npm run typecheck`
   2. `npm run lint`
   3. `npm run build`

## 5) ملاحظات تشغيلية

1. `prisma:seed:core` للبيانات الثابتة المرجعية.
2. `prisma:seed:demo` لبيانات تجريبية قابلة للتغيير.
3. لا تستخدم `seed:demo` في الإنتاج إلا بقصد واضح ومع `ALLOW_PRODUCTION_DEMO_SEED=true`.
