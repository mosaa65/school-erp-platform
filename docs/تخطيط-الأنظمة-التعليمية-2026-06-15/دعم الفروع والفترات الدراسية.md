# دعم الفروع والفترات الدراسية

تاريخ الإعداد: 2026-06-15

## الهدف

هذا الملف منفصل عن خطة تقسيم الأنظمة، لأنه يشرح قاعدة عامة تؤثر على كل الأنظمة التعليمية:

- دعم الفروع المتعددة.
- دعم الفترة الصباحية والمسائية.
- منع تكرار `branchId` في كل جدول بلا حاجة.

## الهيكل المستهدف

```text
الفرع
  -> الفترة الدراسية
      -> الصف
          -> الشعبة
              -> تسجيل الطالب
```

مثال:

```text
فرع صنعاء
  -> صباحي
      -> أول ثانوي
          -> الشعبة أ
```

ومثال آخر:

```text
فرع تعز
  -> مسائي
      -> أول ثانوي
          -> الشعبة أ
```

الشعبة أ في صنعاء ليست نفس الشعبة أ في تعز، وليست نفس الشعبة أ في فترة مسائية إذا كانت المدرسة تفصل الشعب حسب الفترة.

## لماذا هذا مهم؟

بدون هذا التصميم سيحدث خلط في:

- قوائم الشعب.
- حضور الطلاب.
- الواجبات.
- الاختبارات.
- المحصلة الشهرية.
- النتائج.
- صلاحيات المعلمين والمشرفين.

## قاعدة الطالب

الطالب لا يرتبط مباشرة بالفرع والفترة والشعبة داخل ملفه الأساسي.

الطالب يرتبط عبر التسجيل الأكاديمي:

```text
Student
  -> StudentEnrollment
      -> الصف
      -> الشعبة
          -> الفترة الدراسية
              -> الفرع
```

هذا يسمح بأن:

- ينتقل الطالب من شعبة إلى شعبة.
- ينتقل الطالب من فترة إلى فترة إذا سمحت المدرسة.
- يبقى تاريخ الطالب الأكاديمي قابلًا للتتبع.

## قاعدة `branchId`

لا نضيف `branchId` في كل جدول إذا كان يمكن استنتاجه من الشعبة أو تسجيل الطالب.

لا يحتاج `branchId` مباشرًا:

- حضور الطالب.
- واجب الطالب.
- درجة الاختبار.
- المحصلة الشهرية.
- النتيجة الفصلية.
- النتيجة النهائية.
- السلوك والمشاركة.

يمكن أن يحتاج `branchId` مباشرًا:

- المالية.
- المخزون.
- المكتبة إذا كان المخزون مستقلًا حسب الفرع.
- النقل.
- الأصول.

## قاعدة الفترة الدراسية

الفترة الدراسية هنا تعني:

- صباحي.
- مسائي.
- أي فترة تشغيل أكاديمية تعتمدها المدرسة.

ولا تعني:

- الحصة الأولى.
- الحصة الثانية.
- فترات الجدول اليومية.

لذلك يجب عدم خلط:

```text
AcademicShift = صباحي / مسائي
```

مع:

```text
TimetablePeriod = الحصة الأولى / الثانية / الثالثة
```

## أثرها على الأنظمة

### الطلاب والتسجيل

- التسجيل يحدد الصف والشعبة.
- الشعبة تحدد الفترة والفرع.

### الجدول الدراسي

- الجدول يجب أن يعمل داخل شعبة محددة.
- الشعبة تحمل سياق الفترة والفرع.

### الواجبات والاختبارات

- الواجب أو الاختبار يستهدف شعبة أو مجموعة طلاب.
- لا يحتاجان فرعًا مباشرًا إذا كانت الشعبة تحدده.

### المحصلة والنتائج

- النتيجة ترتبط بتسجيل الطالب.
- تسجيل الطالب يكفي للوصول إلى الصف والشعبة والفرع.

## قرار التصميم

نضيف مفهوم الفترة الدراسية ككيان مستقل عند تنفيذ دعم الفروع الكامل.

الاسم المقترح:

```text
AcademicShift
```

أو إذا أردنا ربطها بالفرع مباشرة:

```text
BranchAcademicShift
```

القرار النهائي يعتمد على هل الفترة الصباحية/المسائية عامة لكل الفروع أم تختلف إعداداتها من فرع إلى فرع.

## قاعدة التسمية في الواجهات

عند عرض الشعبة للمستخدم، لا تعرض:

```text
الشعبة أ
```

فقط.

الأفضل عرض:

```text
فرع صنعاء / صباحي / أول ثانوي / الشعبة أ
```

أو في النسخة المختصرة:

```text
أول ثانوي - أ - صباحي - صنعاء
```

هذا يمنع اللبس عند وجود أكثر من فرع أو فترة.
